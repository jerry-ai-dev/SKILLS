# 后训练理论深化教学（post-training-teacher SKILL）

> 一个把 RL / PPO / GRPO / RLHF / SFT 讲到"能落地做项目"的 AI 教学 Skill。
> 面向已经学完 PyTorch 基础、想真正理解 LLM 后训练的人。

---

## 写在最前面：为什么我要做这套 AI 教学 Skill

学大模型算法这一年多，我把能试的路子基本都试了一遍：买书、刷视频、啃博客、硬读源码。
但几样东西都有各自的别扭——书翻两页就开始走神，视频拖进度拖到烦，博客经常讲着讲着就跳了一步关键推导，源码更不用说，一上来就是一个几千行的 `trainer.py`。
最难受的是——**我卡住的时候，它们谁都不会停下来换个说法**。一个公式没看懂，书不会换个类比，视频不会看着我的表情把速度放慢。

真正让我学进去的转折点，是开始**让大模型当我的老师**。

用下来几个月，我的感受是：

- 公式、符号、定义这些通用知识，它讲得**够准，也够细**，基本不会出错；
- 同一个概念可以让它换三四种讲法——打比方、画图、写段 mini 代码、换个角度再来一遍；
- 最爽的是**它不嫌你烦**。你说"还是没懂"，它就重新组织一遍；你说"太简单了"，它就加深一层。

所以我的结论挺简单的：**如果你想系统学后训练，让 AI 教你，大概率是现在最划算的方式。**

不过话说回来，直接丢给 Copilot 一句"教我 GRPO"也是不够用的——它会讲，但讲得散、讲得随缘、没有主线，今天和明天讲的还可能对不上。
所以我干脆把自己学过一遍的经验沉淀成了这套 Skill，主要做了三件事：

1. **把大纲定死**：后训练要学的东西其实没那么多，就是 RL → PPO → RLHF → GRPO → SFT → R1 这条主线。我把它筛成 10 节课，不贪多。
2. **把讲法写死**：每节课 9 步流程、5 种可选风格、彩蛋复盘、复习模式——让 AI 老师每次的产出都稳定，不会今天认真明天摆烂。
3. **每节课我都亲自学过一遍**：这个仓库里的所有 Skill（`pytorch-teacher`、`post-training-teacher`、`code-reading-teacher`…）都是我自己先学一遍、边学边改教案，碰到的坑、想出来的类比、算明白的公式，全都回填进课件里，**确认自己真的学通了才上传**。所以每一节课、每一个彩蛋、每一份复习八股，都是过过脑子的，不是 AI 一键生成的。

### 给你一句实话

后训练这部分，**是真的有点难**。公式密、概念抽象，一节课里新术语堆成山，第一次接触头晕是很正常的事情。
但好消息是——这套 Skill 专门就是为了"头晕"这件事准备的：

- **学不动就降难度。** 5 种讲课风格随便切：先用 A（纯打比方、零公式）把直觉立住，二刷换 C 把公式补上，三刷想抠细节再上 B，想直接看代码就换 D。**一节课刷三遍不丢人，反而最快。**
- **学过 ≠ 记住。** 每节课都配了复习八股文，阶段还有考试，全部学完还能用复习模式再串一遍，把散的知识连成网。
- **前置门槛不高**：有点深度学习 / PyTorch 基础就能上车。**完全零基础？** 先去仓库里的 `pytorch-teacher` Skill 把第一阶段过一遍。

具体怎么用好这些技巧，后面「三、使用技巧」一节会一个一个展开讲。

### 顺便说下模型的选择

体验这套 Skill 的效果，其实和你挂的模型也挺有关系的。我自己试下来：

- **Claude Opus 4.6**：我主力用的就是这个，体验最顺——讲解有层次，类比贴切，公式也不会瞎编，切换风格的时候"人味"最足。**强烈推荐。**
- **Claude Sonnet 4.6**：勉强能打，讲得没 Opus 细，但速度快、便宜，赶进度的时候可以用。
- **GPT-5.4**：能讲对，但**话特别多**，经常一个概念能铺三屏，读起来有点累。

总之"最佳体验"这件事还是和模型强相关的。大家可以自己在 VS Code / Copilot 里切换不同模型跑一节课感受下，挑一个顺手的。**我个人最推荐 Claude Opus 4.6。**

## 一、这个 Skill 在整个学习路线里的位置

笔者为自己规划了一条**从零到能做后训练项目**的完整路线，共分 4 个阶段：

| 阶段 | 名称 | 目标 | 对应 Skill / 产出 |
|------|------|------|-------------------|
| 第一步 | **PyTorch 入门** | 打牢 Tensor / Autograd / Transformer / 微调基础 | `pytorch-teacher` Skill |
| 第二步 | **后训练理论深化**（👈 当前 Skill） | 把 RL、PPO、GRPO、RLHF、SFT 的原理、公式、代码全部搞懂，最终读透 DeepSeek R1 | `post-training-teacher` Skill |
| 第三步 | **开源项目研读** | 系统阅读 TRL、Open-R1、SimpleRL-Zoo 三个项目的源码，理解工业级 pipeline | `code-reading` Skill（待进行） |
| 第四步 | **完成一个后训练项目** | 亲手跑一遍 SFT + GRPO 的完整 pipeline，把前三阶段所学真正落到训练脚本里 | 个人 repo（待进行） |

### 4 个阶段的最终目标

**走完 4 个阶段后，你会具备做一个完整后训练项目的能力。** 对应到求职 JD 里，只要写着：

> "要求有强化学习 / RLHF 经验、熟悉 SFT 流程、理解 PPO/GRPO……"

## 二、本 Skill 学什么：课程大纲

共 **10 节正课 + 3 次考试**，分 3 个阶段。完整大纲见 [references/curriculum.md](references/curriculum.md)。

### 阶段一：RL 理论基础（Lesson 1–3 + Exam 1）
- **Lesson 1** 强化学习基础 & MDP
- **Lesson 2** Policy Gradient & REINFORCE
- **Lesson 3** PPO 算法深入（Importance Sampling / Clip / GAE / Actor-Critic）
- **Exam 1** RL 理论阶段考试

### 阶段二：RLHF 完整流程（Lesson 4–7 + Exam 2）
- **Lesson 4** Reward Model（Bradley-Terry / 偏好损失 / Reward Hacking）
- **Lesson 5** RLHF 三阶段全流程（SFT → RM → RL + KL 惩罚）
- **Lesson 6** GRPO 算法（组内相对奖励，DeepSeek R1 用的那个）
- **Lesson 7** SFT 工程实践（Loss Masking / Packing / `trl.SFTTrainer`）
- **Exam 2** RLHF 阶段考试

### 阶段三：工程与前沿（Lesson 8–10 + Exam 3）
- **Lesson 8** 梯度累积 & 混合精度训练（bf16 / fp16 / GradScaler）
- **Lesson 9** BPE Tokenizer 原理
- **Lesson 10** DeepSeek R1 论文精读（R1-Zero 的 Aha Moment）
- **Exam 3** 期末综合考试（颁发成绩单）

### 标准学习流程：学习 → 考试

每节课都严格按照 **9 步流程** 推进，核心骨架是：

```
检查进度 → 选教学风格 → 前置知识摸底 → 知识讲解
   → 🎁 彩蛋串讲 → 代码演示 → 代码答疑
   → 测验出题 → 判题讲解 → 打卡 & 预告
```

阶段结束有考试课（Exam 1 / 2 / 3），走**出题 → 批卷 → 薄弱点诊断 → 错题讲解 → 针对性复习**的闭环。
详细流程定义见 [SKILL.md](SKILL.md)。

## 三、使用技巧（笔者踩坑总结）

### 技巧 1：5 种教学风格任选——**学不动就降难度，多刷几遍**

说实话，笔者自己第一次学这些内容的时候**觉得太难了**：PPO、GRPO、GAE 的公式一屏一屏地糊脸，一节课下来经常是"字都认识，凑一起就不懂"。硬啃公式只会越学越劝退。
于是笔者想到一个策略——**不要一次就追求完全吃透，允许自己降低难度、分多遍学**。这也就是 5 种教学风格诞生的原因：

| 风格 | 名称 | 公式量 | 什么时候用 |
|------|------|--------|------------|
| **A** | 形象比喻型 | **无** | 🆘 觉得很吃力？先用这个！全程类比讲故事，**只追求听懂直觉**，不碰任何公式 |
| **B** | 硬核原理型 | 多 | 已经懂了直觉，想把数学从头推一遍 / 准备面试推导题 |
| **C** | 折中直觉型（默认）| 中（带人话解释）| 二刷用这个：先给直觉再亮公式，每个公式配一句人话 |
| **D** | 工程实践型 | 少（只作注释）| 概念都懂了，只想看代码怎么落地 |
| **E** | 苏格拉底追问型 | 按需 | 想检验自己理解程度，让老师反过来考你 |

#### 💡 核心用法：**同一节课可以用不同风格反复学**

这个 Skill 完全支持你**一节课刷好几遍、每遍换个风格**：

- **第一遍吃力？** → 用 **风格 A** 过一遍，只求"哦～原来它在干这个"的直觉图景，公式一个不看。
- **有直觉之后二刷** → 换 **风格 C**，这时候再看公式，你会发现"咦？没想象中那么可怕，就是把刚才那个直觉写成了数学"。
- **想更深挖** → 三刷用 **风格 B**，把每一步推导亲手过一遍。
- **准备动手写代码** → 再来一次 **风格 D**，跳过理论直接盯着代码看。
- **想自测** → 最后换 **风格 E**，让老师拿问题反过来考你。

👉 **记住一句话**：**学不动 ≠ 你笨，只是风格不对**。先降到风格 A，把直觉立住；再一层层加公式、加代码。一节课学三遍不丢人，反而是最快的路径。

### 技巧 2：每节课都配套"复习八股文"，课后 5 分钟回忆全部要点

每上完一节 `lessonNN` 正课，`references/` 目录下都会有一份对应的 [`reviewNN_xxx.md`](references/)：
- 核心概念速查表
- 生动类比 & 记忆钩子
- 一句话总结 + 自测题

**打卡环节结束时老师会主动提醒你去翻这份文件**——地铁上、午休时拿出来扫一眼，比回头重刷整节课高效得多。

### 技巧 3：每节课知识讲完后有一个"🎁 彩蛋小剧场"

这是 Step 3.5 的固定环节。在进入代码之前，老师会用**一个完全生活化的故事**（招聘面试、国考阅卷、厨师学徒、打油诗、顺口溜……）把本节所有核心概念**一次性串成一个画面**。
目的只有一个：**让你的大脑在高强度理论后喘口气，同时把零散碎片拼成完整图景**。笔者自测下来，彩蛋是整节课记得最牢的部分。

### 技巧 4：复习模式（Review Mode）⭐ 随时可触发

笔者本人在学完全部 10 节课后，用这个模式**从头到尾又过了一遍**，效果出奇地好——因为此时全部概念都已经存在脑子里，只差一张"网"把它们连起来。

**触发方式**（对老师说任一即可）：
> - "复习" / "再过一遍" / "从头复习"
> - "把后训练过一遍" / "复习后训练"
> - "review" / "post-training review"
> - "我都上完了，带我再过一遍" / "不讲代码只讲理念"

**复习模式的特点**：
- ❌ 不讲复杂推导、不写代码、不创建练习文件、不改动进度
- ✅ 只讲**理念、直觉、因果链**——把散落的概念串成知识网
- ✅ 每站 = 灵魂一句话 + 核心理念 3~5 条 + 承上启下 + 自测三连
- ✅ 节奏是正式课的 **3~5 倍**，可以一口气冲到 Lesson 10 的"毕业串讲"
- ✅ 只提供 **A / C** 两种风格（推荐 C）

**最佳使用时机**：
1. 学完全部 10 课后做一次系统复习（笔者强烈推荐）
2. 考试前一晚过一遍薄弱章节
3. 读开源代码（阶段三）或做项目（阶段四）时回来查概念

## 四、快速开始

对 GitHub Copilot 说任一触发词即可开课：

> - **开始学习**：`开始后训练` / `阶段二` / `学习 PPO` / `学习 GRPO` / `学习 RLHF`
> - **继续学习**：`继续后训练` / `下一课`
> - **复习模式**：`复习后训练` / `把后训练过一遍` / `post-training review`

老师会自动：
1. 读取 [SKILL.md](SKILL.md) 加载教学规范
2. 运行 `scripts/progress.py show` 查看你的进度
3. 从上次的位置继续，或进入复习模式
