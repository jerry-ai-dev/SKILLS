# 后训练理论深化 - 各课前置知识表

每课开始前，教师展示本课涉及的专业概念，让学生自选需要讲解的部分。

## Lesson 1: 强化学习基础 & MDP

| 概念 | 难度 | 一句话简介 |
|------|------|-----------|
| 马尔可夫性质 | ⭐ | 下一状态只取决于当前状态，与历史无关 |
| 期望值 $\mathbb{E}[\cdot]$ | ⭐ | 对随机变量的加权平均 |
| 折扣因子 $\gamma$ | ⭐ | 未来奖励的衰减系数，0 到 1 之间 |
| 监督学习 vs 强化学习 | ⭐ | 前者有标准答案，后者靠奖励信号摸索 |

## Lesson 2: Policy Gradient & REINFORCE

| 概念 | 难度 | 一句话简介 |
|------|------|-----------|
| 对数导数技巧 | ⭐⭐⭐ | $\nabla P = P \cdot \nabla\log P$，让期望可微 |
| Monte Carlo 估计 | ⭐⭐ | 用大量随机样本估计期望值 |
| 梯度上升 vs 梯度下降 | ⭐ | 最大化目标时用梯度上升，等价于对负目标梯度下降 |
| 方差 (Variance) | ⭐⭐ | 随机变量偏离均值的程度，高方差 = 估计不稳定 |

## Lesson 3: PPO 算法深入

| 概念 | 难度 | 一句话简介 |
|------|------|-----------|
| 重要性采样 | ⭐⭐⭐ | 用旧策略数据估计新策略的期望，通过权重比率修正偏差 |
| KL 散度 | ⭐⭐ | 衡量两个概率分布差异的量 |
| Actor-Critic 架构 | ⭐⭐ | 策略网络(Actor) + 价值网络(Critic) 同时训练 |
| 偏差-方差权衡 | ⭐⭐ | 低偏差高方差 vs 高偏差低方差，统计学核心权衡 |

## Lesson 4: Reward Model

| 概念 | 难度 | 一句话简介 |
|------|------|-----------|
| Sigmoid 函数 | ⭐ | 将实数映射到 (0,1) 的函数：$\sigma(x) = 1/(1+e^{-x})$ |
| 二分类 Cross-Entropy | ⭐⭐ | $-y\log\hat{y} - (1-y)\log(1-\hat{y})$，偏好排序的基础 |
| 预训练 LLM 的 backbone | ⭐ | 提取语义特征的核心网络，可在其上加任务头 |
| 过拟合 (Overfitting) | ⭐ | 在训练集上表现好但泛化差，这里特指 RM 过拟合奖励 |

## Lesson 5: RLHF 完整流程

| 概念 | 难度 | 一句话简介 |
|------|------|-----------|
| KL 散度（实践含义）| ⭐⭐ | 在 RLHF 中衡量新策略偏离参考模型多远 |
| 分布偏移 (Distribution Shift) | ⭐⭐ | 训练分布和实际使用分布不同导致性能下降 |
| 三阶段训练流程 | ⭐ | SFT → RM → RL，每阶段有不同的数据和目标 |

## Lesson 6: GRPO

| 概念 | 难度 | 一句话简介 |
|------|------|-----------|
| Baseline 技巧（复习） | ⭐⭐ | 在梯度中减去一个参考值降低方差，不影响均值 |
| z-score 归一化 | ⭐ | $(x - \mu) / \sigma$，使数据均值为0方差为1 |
| 显存占用分析 | ⭐⭐ | 计算训练时各组件的显存消耗 |

## Lesson 7: SFT 工程实践

| 概念 | 难度 | 一句话简介 |
|------|------|-----------|
| Chat Template / ChatML | ⭐ | 规定对话格式的特殊 token 标记方式 |

## Lesson 8: 梯度累积 & 混合精度

| 概念 | 难度 | 一句话简介 |
|------|------|-----------|
| 浮点数表示（fp32/fp16/bf16）| ⭐⭐ | 用有限位数近似表示实数，精度和范围有取舍 |
| 批大小 (Batch Size) 对训练的影响 | ⭐⭐ | 更大 batch 通常梯度更稳定但显存更多 |
| 指数移动平均（Adam 优化器）| ⭐⭐ | Adam 中的一阶和二阶矩估计，fp32 存储防止精度损失 |

| 概念 | 难度 | 一句话简介 |

## Lesson 10: DeepSeek R1 论文精读

| 概念 | 难度 | 一句话简介 |
|------|------|-----------|
| Chain-of-Thought (CoT) | ⭐ | 让模型逐步推理而不是直接给答案的技术 |
| 涌现现象 (Emergent Behavior) | ⭐⭐ | 训练过程中出现训练数据中从未出现的新行为 |
| Rejection Sampling | ⭐⭐ | 从模型采样大量结果，只保留满足条件的，作为新训练数据 |
| Pass@K 评估指标 | ⭐ | 生成 K 个答案，至少一个正确即算通过，衡量推理能力 |
