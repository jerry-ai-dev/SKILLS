## 一、核心概念速查表

| 概念 | 一句话定义 | 生活类比 |
|------|-----------|---------|
| 强化学习 (RL) | 智能体通过与环境交互、获得奖励反馈来学习最优行为 | 小孩学走路：摔了（负奖励）就调整姿势，走稳了（正奖励）就保持 |
| MDP | 用 $(S, A, P, R, \gamma)$ 五元组描述决策问题的数学框架 | 棋盘游戏规则说明书：有哪些局面、能走什么棋、走完会怎样、赢了得多少分 |
| 状态 (State) $s$ | 当前环境的完整描述 | 下棋时的棋盘局面 |
| 动作 (Action) $a$ | 智能体在某状态下的选择 | 在棋盘上落下一子 |
| 奖励 (Reward) $R$ | 执行动作后环境给出的即时反馈分数 | 老师给的当堂小红花 |
| 折扣因子 $\gamma$ | 控制"未来奖励"相对"当前奖励"的重要程度，$\gamma \in [0,1)$ | 今天的 100 块 vs 明年的 100 块——钱有时间价值 |
| 策略 (Policy) $\pi_\theta(a|s)$ | 在状态 $s$ 下选择动作 $a$ 的概率分布 | 你的"做事习惯"——遇到什么情况倾向于怎么做 |
| 回报 (Return) $G_t$ | 从 $t$ 时刻起的折扣累计奖励之和 | 一学期的加权总成绩（期末考试权重比平时作业高） |
| 价值函数 $V^\pi(s)$ | 在状态 $s$ 下，遵循策略 $\pi$ 的期望回报 | "在当前局面下，按你的习惯打下去，期望能赢多少分" |
| Q 函数 $Q^\pi(s,a)$ | 在状态 $s$ 下执行动作 $a$，然后遵循 $\pi$ 的期望回报 | "在这个局面下走这步棋，之后按习惯下完，期望多少分" |

## 二、关键公式卡片

### 回报（轨迹总奖励）
$$G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}$$
> **人话**：把未来的奖励一个个"打折"加起来。$\gamma=0.9$ 意味着 10 步之后的奖励只值当前的 $(0.9)^{10} \approx 0.35$ 倍。

### 价值函数
$$V^\pi(s) = \mathbb{E}_\pi[G_t \mid s_t = s]$$
> **人话**：站在这个路口，按你的习惯走下去，平均能拿多少总分。

### 策略函数（LLM 对应）
$$\pi_\theta(\text{next token} \mid \text{context}) = \text{Softmax}(\text{logits})$$
> **人话**：语言模型每一步的 Softmax 输出就是策略——决定下一个 token 选谁。

## 三、生动类比：火锅店点菜

想象你第一次去一家火锅店，菜单上有 50 道菜（**动作空间**），你对每道菜的好吃程度一无所知。

- **状态**：你目前的饥饿程度 + 已经点了哪些菜
- **动作**：从菜单中选一道新菜
- **奖励**：吃完后的满意度打分（好吃 +1，难吃 -1）
- **策略**：你的点菜偏好（"我倾向于先点肉类"就是一种策略）
- **折扣因子**：第一道菜给你的快乐 > 第十道菜给你的快乐（你越吃越饱）
- **回报**：这顿饭的总满意度
- **探索 vs 利用**：你是试新菜（探索）还是一直点上次好吃的（利用）？

**ε-贪心策略**就像：每次有 10% 概率随便点一道（探索），90% 概率点你目前觉得最好吃的（利用）。

## 四、LLM 中的 RL 对应关系速记

| 火锅店场景 | MDP 概念 | LLM 训练中的含义 |
|-----------|---------|-----------------|
| 菜单上的 50 道菜 | 动作空间 $A$ | 词表中的 3 万~10 万个 token |
| 已经点的菜 + 饥饿度 | 状态 $s_t$ | 当前已生成的 token 序列（上下文） |
| 选一道新菜 | 动作 $a_t$ | 生成下一个 token |
| 吃完打分 | 奖励 $R$ | 整条生成文本的奖励（答案对 +1，错 0） |
| 整顿饭的满意度 | 回报 $G_t$ | 累计折扣奖励 |
| "先点肉类"的习惯 | 策略 $\pi$ | 模型的 Softmax 输出分布 |

---

## 五、易混淆点 & 常见误区

1. **奖励 vs 回报**：奖励 $R$ 是单步的即时反馈，回报 $G_t$ 是从 $t$ 到结束的累计折扣总和。不要混淆！
2. **$\gamma = 0$ vs $\gamma \to 1$**：$\gamma=0$ 只看眼前（短视），$\gamma \to 1$ 远近一视同仁（可能导致回报发散）。
3. **LLM 的奖励是稀疏的**：不同于游戏每步都有分数，LLM 通常只有生成完整回答后才拿到一个分数，这增加了训练难度。
4. **$V(s)$ vs $Q(s,a)$**：$V$ 只看状态，$Q$ 看的是"在这个状态下做了某个具体动作"——$Q$ 多了一个动作维度。

---

## 六、记忆口诀

> **"五元一策总回报，折扣越小越短视"**
> - 五元 = MDP 五元组 $(S, A, P, R, \gamma)$
> - 一策 = 策略 $\pi$
> - 总回报 = $G_t$
> - 折扣越小越短视 = $\gamma$ 小 → 只看近期
