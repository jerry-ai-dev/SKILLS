## 一、核心概念速查表

| 概念 | 一句话定义 | 生活类比 |
|------|-----------|---------|
| 重要性采样 (IS) | 用旧策略采的数据来估计新策略的期望 | 用去年的考试题（旧数据）来预测今年的考试趋势（新分布） |
| 概率比率 $r_t(\theta)$ | $\frac{\pi_\theta(a_t|s_t)}{\pi_{\theta_\text{old}}(a_t|s_t)}$，衡量新旧策略的差异 | 你今年选择这道菜的概率 / 去年选它的概率。>1 说明更爱吃了 |
| PPO-Clip | 用 clip 操作限制策略更新幅度的标函数 | 汽车油门限位器——再怎么踩也只能到某个转速，防止发动机爆缸 |
| Clip 系数 $\epsilon$ | 允许新旧策略概率比率偏离 1 的最大范围（通常 0.1~0.2） | 限速标志写的 ±20km/h——偏离太多就截断 |
| GAE-$\lambda$ | 广义优势估计，用 $\lambda$ 在 MC（高方差）和 TD（高偏差）间权衡 | 期末成绩评估方式：只看期末考试（MC）vs 只看每次小测（TD）vs 加权混合（GAE） |
| TD 误差 $\delta_t$ | $r_t + \gamma V(s_{t+1}) - V(s_t)$，单步的"惊喜度" | "这步实际得分 - 预期得分"，正数 = 意外之喜 |
| Actor-Critic | 策略网络(Actor)决策 + 价值网络(Critic)评分的双网络架构 | 球员(Actor)上场踢球 + 教练(Critic)在场边打分指导 |

## 二、关键公式卡片

### 概率比率
$$r_t(\theta) = \frac{\pi_\theta(a_t|s_t)}{\pi_{\theta_\text{old}}(a_t|s_t)}$$
> **人话**：新策略对这个动作的"热情度"相比旧策略变化了多少。

### PPO-Clip 目标函数
$$\mathcal{L}^{\text{CLIP}} = -\mathbb{E}_t\left[\min\left(r_t \hat{A}_t,\; \text{clip}(r_t, 1-\epsilon, 1+\epsilon) \hat{A}_t\right)\right]$$
> **人话**：取"不限制"和"限制"两个版本中更保守的那个。好动作不能过度强化，坏动作不能过度惩罚。

### GAE-$\lambda$ 递推
$$\hat{A}_t^{\text{GAE}} = \delta_t + \gamma\lambda \cdot \hat{A}_{t+1}$$
> **人话**：当前的优势 = 当前的"惊喜" + 未来打折后的优势。$\lambda$ 控制考虑未来多远。

## 三、生动类比：驾校学车

你正在学开车（**策略 $\pi$**），驾校教练（**Critic**）在副驾驶打分：

**没有 Clip（TRPO 之前）**：
- 你今天练了一次倒车入库，教练说"太棒了"（$\hat{A}>0$）
- 你兴奋过头，下次倒车直接全油门冲进去——结果撞墙了（**策略崩溃**）
- 问题：一次好反馈导致你改变太大

**有 Clip（PPO）**：
- 教练还是说"太棒了"，但你身上有个"限速器"
- 不管教练多激动，你的操作改变不能超过 ±20%（$\epsilon = 0.2$）
- 即使这次倒库完美，你下次也只会稍微加点油门，不会猛冲

**GAE-$\lambda$ 比喻**：
- $\lambda = 0$（只看当堂小测）：教练每一步都打分，分数准但只看当下
- $\lambda = 1$（只看期末考试）：等你整个科目考完再评价，全面但波动大
- $\lambda = 0.95$（加权混合）：既参考每步评价，也考虑最终结果——**最实用**

## 四、Clip 的分类讨论（重点！）

| 情况 | $\hat{A}_t$ | $r_t$ 变化 | Clip 行为 |
|------|------------|-----------|----------|
| 好动作，概率增加 | > 0 | > 1+ε | **截断**：不能再增加了（防止过度强化） |
| 好动作，概率适中 | > 0 | ∈ [1-ε, 1+ε] | 不截断：正常增大概率 |
| 坏动作，概率减少 | < 0 | < 1-ε | **截断**：不能再减少了（防止过度惩罚） |
| 坏动作，概率适中 | < 0 | ∈ [1-ε, 1+ε] | 不截断：正常减小概率 |

> **核心直觉**：Clip 像一根橡皮筋，把新策略拉在旧策略附近，不让它跑太远。

## 五、PPO 完整训练循环速记

```
1. 用旧策略 π_old 采集一批轨迹（play 一局）
2. Critic 估计每步的 V(s)，用 GAE 算优势 Â_t
3. 用同一批数据更新 K 次（重复利用！）：
   a. 算 ratio = π_new / π_old
   b. 算 Clip loss（取 min）
   c. 算 Critic loss（V 预测误差）
   d. 加上熵奖励（鼓励探索）
   e. 反向传播更新 Actor + Critic
4. π_old ← π_new，回到第 1 步
```

## 六、PPO → GRPO 的关键区别

| | PPO | GRPO |
|---|-----|------|
| Critic | 需要（参数量 = Actor） | **不需要**（省一半显存） |
| 优势估计 | GAE（需要 $V(s)$） | 组内奖励归一化（只需 $r_i$） |
| 数据复用 | 同一批数据更新 K 次 | 同理也可多次更新 |
| Clip | ✅ 使用 | ✅ 同样使用 |

> **GRPO 的核心创新**：用"组内比均值"替代 Critic 估计，省显存又简单。

---

## 七、易混淆点 & 常见误区

1. **$r_t$ 是概率比率，不是奖励！** $r_t = \pi_\text{new}/\pi_\text{old}$，和 reward 无关。
2. **为什么取 min？** 取 min 是取更"悲观"的估计——好处不贪多算，坏处不多惩罚。这是 PPO 稳定性的关键。
3. **$\epsilon$ 不是学习率**：$\epsilon$ 是 clip 范围，通常 0.1~0.2，和优化器的 lr 是两码事。
4. **K 次更新不是 K 个 epoch**：是用同一批采样数据做 K 次梯度更新，效率远高于 REINFORCE。

## 八、记忆口诀

> **"比率裁剪不越界，好的别贪坏的别踩；GAE 权衡偏与方，Actor-Critic 一起嗨"**
> - 比率裁剪 = PPO Clip
> - 好的别贪 = 优势 > 0 时 clip 上界
> - 坏的别踩 = 优势 < 0 时 clip 下界
> - GAE = 平衡偏差和方差
> - Actor-Critic = 双网络架构
