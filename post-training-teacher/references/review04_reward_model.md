## 一、核心概念速查表

| 概念 | 一句话定义 | 生活类比 |
|------|-----------|---------|
| Reward Model (RM) | 学习人类偏好、给 (Prompt, Response) 打分的模型 | 美食评委：不做菜，只品尝打分 |
| Bradley-Terry 模型 | 将两个选项的相对偏好建模为分数差的 sigmoid | 两队比赛，实力差越大越可能赢——用 sigmoid 建模胜率 |
| 偏好数据 $(x, y_w, y_l)$ | 一个 Prompt + 胜者回答 + 败者回答 | 同一个菜名，两个厨师做的菜，评委说哪个更好吃 |
| Reward Hacking | 模型找到骗 RM 高分的"捷径"，实际质量没提升 | 学生发现考试只考选择题，就猛猜 C 从不看题——分数高了但没真学会 |
| KL 惩罚 | 限制策略不偏离 SFT 参考模型太远，防止 Reward Hacking | 作文考试的"字迹分"——即使内容有创意，字迹太差也扣分，逼你在老实写字和发挥创意间取平衡 |

## 二、关键公式卡片

### Bradley-Terry 偏好概率
$$P(y_w \succ y_l \mid x) = \sigma(r_\phi(x, y_w) - r_\phi(x, y_l))$$
> **人话**：胜者分数减败者分数，喂进 sigmoid 得到胜率。分差越大，sigmoid 越接近 1，越"有把握"说胜者更好。

### RM 训练 Loss
$$\mathcal{L}_{\text{RM}} = -\mathbb{E}\left[\log\sigma\left(r_\phi(x, y_w) - r_\phi(x, y_l)\right)\right]$$
> **人话**：训练目标就是让胜者分数 >> 败者分数。分差大 → sigmoid 大 → log 接近 0 → loss 小。

### RM 网络结构
```
[Prompt + Response] → LLM Backbone → 最后 token 的隐状态 → Linear(d, 1) → 标量分数 r
```
> **人话**：在一个已有的语言模型尾巴上接一个线性层，输出一个数字作为分数。

## 三、生动类比：选秀节目评委

想象一个歌唱选秀节目：

- **选手 A 和 B**（$y_w$ 和 $y_l$）唱同一首歌（同一个 **Prompt $x$**）
- **评委**（**Reward Model**）不唱歌，只打分
- 评委一开始不知道谁好谁差，需要看**大量对决数据**来训练自己的审美（训练 RM）
- 训练目标：评委给唱得好的人打更高分——**分差越大越好**

**Reward Hacking** 就像：
- 有个选手发现评委特别喜欢高音（RM 的偏见），就全程飙高音
- 声音其实很难听，但评委给了高分——骗到了评委
- **KL 惩罚**就像节目组加的规则："你的唱法不能跟正常唱法差太多，否则直接扣分"

## 四、RM 训练流程速记

```
1. 收集偏好数据：人类标注员对同一 Prompt 的多个回答排序
   → 得到 (prompt, chosen, rejected) 三元组

2. 编码：把 [prompt + chosen] 和 [prompt + rejected] 分别喂入 RM
   → 得到两个标量分数 r_w 和 r_l

3. 计算 loss = -log σ(r_w - r_l)
   → 让 r_w 尽可能大于 r_l

4. 反向传播更新 RM 参数

5. 评估指标：分类准确率（r_w > r_l 的比例）
```

---

## 五、Reward Hacking 的三种防御

| 防御方法 | 原理 | 比喻 |
|---------|------|------|
| KL 散度惩罚 | 策略不能偏离 SFT 模型太远 | 你可以创新，但不能变成另一种物种 |
| 定期更新 RM | 随着策略变化重新训练 RM | 评委也要跟上选手的新风格 |
| 多 RM 取最小分 | 多个评委打分取最低分 | 保守估计——任何一个评委不满意就不算高分 |

## 六、你的项目 vs 标准 RM

| 标准 RLHF | 你的数学推理项目 |
|-----------|----------------|
| 需要训练 RM（标注数据昂贵） | 不需要 RM！用数学验证器直接打分 |
| 奖励可能 hack | 答案对就是对，不存在被骗 |
| 打分维度多（安全、有用、诚实） | 打分维度单一（对/错） |

> **这是 DeepSeek R1 最巧妙的设计之一**：数学题的奖励天然"不可 hack"，所以可以跳过 RM 直接用规则奖励。

## 七、易混淆点 & 常见误区

1. **RM 不是 LLM**：RM 的输出是一个标量分数，不是 token 序列。虽然 backbone 是 LLM，但输出头完全不同。
2. **Bradley-Terry 只看分差**：$\sigma(r_w - r_l)$ 只关心两个分数的差，不关心绝对值。$r_w=10, r_l=8$ 和 $r_w=100, r_l=98$ 效果一样。
3. **chosen/rejected 标反了会怎样？** 模型会学到"差回答得高分"——完全学反。偏好标注质量极其重要。
4. **RM 准确率 ≠ 100%**：即使人类标注员之间也只有 ~70% 的一致性率。RM 是对人类偏好的近似。

## 八、记忆口诀

> **"胜者分高败者低，sigmoid 差值做概率；当心模型来作弊，KL 惩罚拴牢你"**
> - 胜者分高败者低 = $r_w > r_l$
> - sigmoid 差值 = $\sigma(r_w - r_l)$
> - 模型作弊 = Reward Hacking
> - KL 拴牢 = KL 散度惩罚
