## 一、核心概念速查表

| 概念 | 一句话定义 | 生活类比 |
|------|-----------|---------|
| RLHF | 用人类反馈训练 RL，让 LLM 对齐人类偏好 | 老师批改作文 → 总结评分标准 → 学生按标准自我改进 |
| SFT（阶段 1） | 用专家示范数据做监督微调，让模型学会基本格式 | 新厨师先照菜谱学做菜，掌握基本功 |
| RM（阶段 2） | 从人类偏好对中训练打分模型 | 请一批美食家品尝对比，训练出一个"AI 美食评委" |
| RL/PPO（阶段 3） | 用 RM 的分数作为奖励信号优化策略 | 厨师按 AI 评委的打分反馈改进自己的菜 |
| KL 散度 | 衡量两个概率分布之间的差异 | 你的新做菜风格跟传统菜谱的"偏离度" |
| $\beta$ (KL 系数) | 控制策略偏离参考模型的惩罚强度 | 限制厨师的创新幅度——$\beta$ 大 = 保守创新，$\beta$ 小 = 大胆创新 |
| $\pi_\text{ref}$ | 冻结的 SFT 参考模型 | 传统菜谱（标准线） |
| InstructGPT | 第一个大规模 RLHF LLM（OpenAI, 2022） | RLHF 技术的"开山之作" |

## 二、关键公式卡片

### RLHF 阶段 3 优化目标
$$\max_\theta \mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\theta}\left[r_\phi(x, y) - \beta \cdot \text{KL}(\pi_\theta(y|x) \| \pi_\text{ref}(y|x))\right]$$
> **人话**：让奖励分数尽量高，但不能跟 SFT 模型差太多。奖励 vs 保守——$\beta$ 控制天平的倾斜方向。

### 有效奖励
$$r_{\text{effective}} = r_\phi(x, y) - \beta \cdot \log\frac{\pi_\theta(y|x)}{\pi_\text{ref}(y|x)}$$
> **人话**：最终奖励 = RM 分数 - 偏离惩罚。偏离越大，惩罚越重。

### KL 散度
$$\text{KL}(\pi_\theta \| \pi_\text{ref}) = \sum_{y} \pi_\theta(y|x) \log\frac{\pi_\theta(y|x)}{\pi_\text{ref}(y|x)}$$
> **人话**：新分布和旧分布有多"不像"。完全一样 → KL=0，非常不同 → KL 很大。

## 三、生动类比：作文教学三步走

**阶段 1 — SFT（抄范文阶段）**：
- 老师发 50 篇**范文**，学生照抄学习
- 学会了作文的基本格式：开头、论点、结尾
- 输出：一个"能写规范作文"的学生（$\pi_\text{SFT}$）

**阶段 2 — RM（训练评委阶段）**：
- 让真正的语文老师看 1000 对作文，说"A 比 B 好"
- 用这些偏好数据训练一个 **AI 批改老师**（RM）
- 以后就不用真老师每次亲自批改了

**阶段 3 — RL/PPO（独立练习阶段）**：
- 学生自己写作文，AI 老师自动打分
- 高分作文的写法被"强化"，低分写法被"压制"
- 但有一条底线：**不能写得太离谱**（KL 惩罚）——如果你的作文完全不像正常作文了，即使 AI 老师打高分也扣分

**KL 惩罚 $\beta$ 的作用**：
- $\beta$ 大 = 老师说"别太创新，规规矩矩写"
- $\beta$ 小 = 老师说"大胆想，但别太离谱"

## 四、RLHF 三阶段速记表

| 阶段 | 输入数据 | 训练目标 | 输出 |
|------|---------|---------|------|
| 1. SFT | (Prompt, 专家回答) | Cross-Entropy Loss | $\pi_\text{SFT}$（参考模型 $\pi_\text{ref}$） |
| 2. RM | (Prompt, $y_w$, $y_l$) | $-\log\sigma(r_w - r_l)$ | 奖励模型 $r_\phi$ |
| 3. PPO | Prompt → 采样回答 → RM 打分 | $r_\phi - \beta \cdot \text{KL}$ | 对齐后的 $\pi_\theta^*$ |

> **顺序铁律**：SFT → RM → PPO，不能跳步！每一步都依赖前一步的输出。

## 五、你的项目 vs 标准 RLHF

| 标准 RLHF | 你的数学推理项目 |
|-----------|----------------|
| 人类偏好标注 RM | 规则奖励（答案对 +1，错 0） |
| PPO + Critic | GRPO（无 Critic） |
| 对齐安全/有用/诚实 | 提升推理正确率 |
| 需要 RM + 大量标注 | 不需要 RM，实现简单 |
| KL 惩罚 ✅ | KL 惩罚 ✅（同样需要） |

> **省事在哪**：不用标注偏好数据，不用训练 RM，不用维护 Critic。但核心算法思路完全一致。

## 六、易混淆点 & 常见误区

1. **SFT ≠ 预训练**：预训练用海量文本学"语言能力"，SFT 用少量指令对学"行为规范"。
2. **$\beta$ 太大的后果**：模型几乎不更新，等于白训。$\beta$ 太小：模型疯狂优化 RM 分数，可能 hack。需要调到平衡。
3. **RM 的准确率上限**：人类标注者之间的一致率约 70%，所以 RM 理论上也难超过这个水平。
4. **KL 散度不对称**：$\text{KL}(\pi_\theta \| \pi_\text{ref}) \neq \text{KL}(\pi_\text{ref} \| \pi_\theta)$。RLHF 用的是前者（前向 KL）。

## 七、记忆口诀

> **"先抄作文学格式，再请评委训打分，最后自练求高分，KL 套牢不跑偏"**
> - 抄作文 = SFT
> - 训打分 = 训练 RM
> - 自练 = PPO/GRPO
> - KL 套牢 = KL 惩罚
