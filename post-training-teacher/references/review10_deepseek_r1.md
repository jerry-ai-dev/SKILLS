## 一、核心概念速查表

| 概念 | 一句话定义 | 生活类比 |
|------|-----------|---------|
| R1-Zero | 仅用 RL（无 SFT）就让模型涌现推理能力的实验 | 不给菜谱，只告诉厨师"做得好吃加分"——厨师自己摸索出了名菜 |
| Aha Moment | 模型在训练中自发涌现"自我反思"行为 | 学生自己学会了"等一下，让我重新想想"——没人教过这个 |
| 规则奖励 | 用验证器（而非 RM）直接检查答案对错 | 数学老师只看最终答案："360 对了 +1 分，350 错了 0 分" |
| 格式奖励 | 检查输出是否使用了指定标签格式 | 考试必须写在答题卡上，写在草稿纸上不算 |
| Cold Start SFT | 用少量高质量 CoT 数据做初始微调 | 先教新厨师基本的菜谱格式，再让他自由发挥 |
| Rejection Sampling | 用 RL 模型生成大量回答，只保留正确且格式好的做新一轮 SFT | 千里挑一：生成 100 道菜只留下最好的 5 道写进新菜谱 |

## 二、DeepSeek R1 四阶段流程速记

```
阶段 1: Cold Start SFT
  → 少量高质量 CoT 数据做 SFT
  → 让模型学会 <think>...</think><answer>...</answer> 格式

阶段 2: GRPO 推理增强（RL 第一轮）
  → 用 GRPO + 规则奖励，专攻数学/代码
  → 推理能力大幅提升，但通用能力可能退化

阶段 3: Rejection Sampling + SFT
  → 用阶段 2 模型生成大量回答，筛选正确的
  → 结合通用对话数据做 SFT，恢复通用能力

阶段 4: GRPO 全面对齐（RL 第二轮）
  → 同时优化推理 + 对话 + 安全性
  → 产出：最终 DeepSeek R1 模型
```

## 三、生动类比：培养全能厨师

**阶段 1（Cold Start SFT）**——学徒入门：
- 给厨师 100 份精选菜谱，教他标准做菜流程
- 重点不在于菜的创新，而是**格式规范**（先切菜、再焯水、最后装盘）

**阶段 2（GRPO 第一轮）**——专项特训：
- 只给一个评分标准："味道好不好吃"（**规则奖励**）
- 同一道菜让厨师做 16 次（**G=16**），好的强化坏的压制
- 厨师数学突飞猛进，但可能忘了怎么做西餐（**通用能力退化**）

**阶段 3（Rejection Sampling + SFT）**——精选菜谱2.0：
- 让特训后的厨师做 10000 道菜，只挑出最好的 500 道
- 这 500 道 + 以前的西餐/日料菜谱混合，让厨师重新学
- 效果：数学能力保住了，西餐也恢复了

**阶段 4（GRPO 第二轮）**——全面打磨：
- 评分标准扩展：味道 + 卫生 + 服务态度 + ...
- 最终产出一个全能厨师

## 四、R1-Zero 的"惊人发现"

模型在训练到一定程度后，**自动**出现了这样的输出：

```
<think>
Let me compute 15 × 24...
First: 15 × 24 = 15 × 20 + 15 × 4 = 300 + 60 = 360
Wait, let me verify: 24 × 15 = 24 × 10 + 24 × 5 = 240 + 120 = 360 ✓
</think>
<answer>360</answer>
```

**"Wait, let me verify"** 完全是模型自己学会的！训练数据中没有任何示例包含这种自我纠正行为。

> **为什么会涌现？** 因为"验证后修正"能提高答题正确率 → 正确率高 → 奖励高 → 这种行为被强化。RL 只要求结果对，但模型自己发现了"验证"这个提高结果的策略。

## 五、R1 vs InstructGPT 终极对比

| 维度 | InstructGPT / ChatGPT | DeepSeek R1 |
|------|----------------------|-------------|
| 目标 | 对齐价值观/安全 | 增强推理能力 |
| 奖励来源 | 人类标注 → Reward Model | 规则验证器（答案对不对） |
| RL 算法 | PPO | GRPO |
| 需要 RM？ | ✅（大工程） | ❌（简化极多） |
| 需要 Critic？ | ✅（PPO 要求） | ❌（GRPO 不需要） |
| 训练阶段 | 3 阶段 | 4 阶段 |
| 涌现行为 | 指令跟随 | 自我反思推理 |
| 数据成本 | 高（人工标注偏好） | 低（规则自动打分） |

## 六、你的项目 = R1 的简化版

```
R1 完整版:  SFT → GRPO → Rejection Sampling + SFT → GRPO
你的项目:   SFT → GRPO
             ↑       ↑
          阶段1     阶段2
```

**你要做的**：
- 用数学 QA 数据做 SFT（让模型学会格式）
- 用 GRPO + 答案正确性奖励训练（让模型学会推理）

**你不需要做的**：
- Rejection Sampling（阶段 3）——小模型没有"通用能力退化"的严重问题
- 全面对齐（阶段 4）——你的目标单纯是数学推理
- 训练 RM——直接用规则奖励

## 七、GRPO 关键超参数（R1 使用的值）

| 超参数 | R1 的值 | 含义 |
|--------|--------|------|
| G | 16 | 每个 prompt 生成 16 条候选回答 |
| $\epsilon$ | 0.2 | PPO clip 范围 |
| $\beta$ | 0.1 | KL 惩罚系数 |
| max_seq_len | 32K | 思维链可以非常长 |

## 八、易混淆点 & 常见误区

1. **R1 ≠ R1-Zero**：R1-Zero 是纯 RL 实验（没有 SFT），R1 是完整四阶段产品。R1 质量更好但 R1-Zero 的发现更有学术价值。
2. **规则奖励不等于简单**：数学验证器要处理各种格式（分数、根号、不同写法的等价答案），实际工程量不小。
3. **涌现不是随机的**：有因果逻辑——验证行为提高正确率 → 正确率提高奖励 → 奖励强化验证行为。
4. **4 阶段中只有 2 阶段用 RL**：阶段 1 和阶段 3 是 SFT（监督学习），阶段 2 和阶段 4 才是 RL。
